Local Trading SDK
with signed daemon IPC

Important: this software is experimental and not production-ready. It is published publicly only for transparency and security audit. Do not rely on it to protect funds, credentials, or sensitive data in real trading environments.
UNIX socket IPC
Signed daemon responses (ed25519)
BoltDB storage (encrypted records)
Local UI (WebSocket → daemon)

Components

cmd/traderd · cmd/ui

traderd runs locally and exposes a minimal HTTP API over a UNIX socket. ui serves the settings page and bridges browser WebSocket messages to traderd.

Signed IPC

X-Req-Id · X-Signature

Signed endpoints include headers X-Req-Id and X-Signature. Clients verify ed25519 over reqID + "\\n" + rawBody.

Web UI Protocol

Legacy WS

Browser protocol stays stable: {type:"list"} / {type:"save"}. The UI server maps exchanges to daemon records internally.

Quickstart

Go 1.25+
# clone
git clone git@github.com:magma-trading/magma-sdk.git
cd magma-sdk

# build
go build -o ./bin/traderd ./cmd/traderd
go build -o ./bin/ui     ./cmd/ui

# run daemon (example flags may differ by your cmd/traderd)
./bin/traderd -sock /tmp/traderd.sock -db ./traderd.db -allowlist-file ./allowlist.txt

# run UI
./bin/ui -addr localhost:8020 -traderd-sock /tmp/traderd.sock

# open
# http://localhost:8020/
